# UI Factory Product Context

**Register:** Product

## Product purpose

UI Factory helps a project owner compare polished single-page design directions, inspect one in context, and export a trustworthy `DESIGN.md` for implementation agents. The durable product model and scope live in [the product vision](ui-factory-product-vision.md).

## Users

### Primary: project owner

Needs to choose a visual direction quickly and confidently without creating an account or decoding implementation details.

### Secondary: design maintainer

Needs to publish, review, and evolve independent entries without changing the catalog application for each addition.

### Downstream: implementation agent

Consumes the selected entry's self-contained `DESIGN.md`. The export, not the browsing interface, is the agent-facing product artifact.

## Core job

Make visual comparison easy, make quality and intended use legible, and make the selected handoff unambiguous.

## Product experience

- **Register:** A focused browsing tool; design serves evaluation and selection.
- **Tone:** Calm, precise, candid, and concise.
- **Information priority:** Preview first, identity and fit second, metadata third, export action when qualified.
- **Trust signals:** Stable identity, explicit quality status, representative responsive behavior, clear rationale, and exact exports.
- **Empty state:** Explain that no designs are published without inventing content or presenting unavailable controls.

## Strategic principles

1. Keep human visual selection in control.
2. Optimize for meaningful visual difference, not catalog size.
3. Keep each design independent from the factory's visual system.
4. Export curated files exactly as published; do not synthesize them on demand.
5. Reveal detail progressively so the catalog remains easy to scan.
6. Prefer native platform behavior and add complexity only after demonstrated need.
7. Treat accessibility, responsive behavior, interaction states, and honest status as publication requirements.

## Anti-references

UI Factory should not resemble:

- A template marketplace dominated by promotion, pricing, and badges
- An administration dashboard filled with decorative metrics
- A generative prompt interface or visual page builder
- Component-library documentation organized around isolated widgets
- A mood board that hides implementation readiness and trade-offs
- A gallery where every entry inherits the factory's own visual language
